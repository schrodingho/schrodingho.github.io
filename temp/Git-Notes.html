Detailed Instruction

创建一个版本库

  • cd进入一个目录, 并mkdir
  • git init 可以把目录 变为 git 可管理的仓库
  • git add (eg: git add readme.txt) 将readme.txt添加到暂存区
  • git commit (eg: git commit -m ‘提交注释’) 提交文件到repository
  • git status 查看是否还有文件未提交,若出现“未被提交的修改”，则可以使用 git diff readme.txt 查看文件到底改了什么内容

版本回退

  • git log 查看修改历史记录
    • 简化版: git log -pretty=oneline
  • git reset –hard HEAD^ 恢复至上一个版本
  • git checkout –file 丢弃工作区的修改. 把file文件在工作区做的修改全部撤销, 会产生两种情况:
    • readme.txt自动修改后，还没有放到暂存区，使用 撤销修改就回到和版本库一模一样的状态。
    • 另外一种是readme.txt已经放入暂存区了，接着又作了修改，撤销修改就回到添加暂存区后的状态。

删除文件

两种一般方法

  • 手动删除文件，add到暂存区，再commit掉
  • git reset –hard HEAD^ 恢复到上一个版本
