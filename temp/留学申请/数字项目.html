精简指令集16位RISC CPU设计

实验目的

  • 掌握处理器 CPU 的结构组成、模块划分和工作原理
  • 掌握精简指令集的概念，并设计 CPU 的指令集系统
  • 掌握精简指令集的概念，并设计 CPU 的指令集系统
  • 掌握激励文件的编写方法，学会验证各个模块功能时序的正确性
  • 掌握外围接口电路的设计方法，并学会利用开发板的按键、LED 等外设构造
    简单的控制系统
  • 编写汇编程序，并依据设计的指令集编译为对应的机器码，控制 CPU 实现特
    定功能

    实验内容

    基于对精简指令集 CPU 的 FPGA 设计示例的学习，理解 cpu 结构，工作流程，完成精简指令集 CPU 的方案设计；理解并掌握利用 verilog 语言在 FPGA 平台上实现微处理器的设计、仿真和下板验证全流程，根据设计方案使用 verilog 语言实现CPU 各模块的建模；编写各模块及 CPU 的激励文件，并对各模块及 CPU 进行仿真验证；编写 CPU 模块并汇总成为一个完整的 CPU，最大限度还原自己的设计方案，通过软件仿真，实现自己设计的精简指令集系统；基于自行设计的指令系统编写汇编程序，参照指令集编译为机器码，使自己设计的 CPU 控制 DE1 板载资源实现特
    定功能。用示波器测试FPGA引脚上的信号，观测信号波形；用逻辑分析仪测试FPGA引脚上的 CPU 总线信号，观测信号时序

    重要信息

    CPU架构

    CPU 为冯.诺依曼结构，使用了 16 位精简指令集，内部的数据路径、寄存器、地址均是 16 位，对存储器可以寻址 64M 的存储空间。对于 IO 口可以寻址2^8的存储空间

    CPU电路

  • 数据通路部分主要由以下基本部件组成：寻址单元（由程序计数器和地址逻辑组成）、算术逻辑单元、通用寄存器阵列、指令寄存器、状态寄存器。
  • 控制器：控制器是 7 状态的状态机，不同状态下向数据通路发送不同的控制信
    号。控制器采用 Huffman 风格的编码，通过把下一状态值赋给 reg 类型的变量 Nstate,来实现状态转化。
